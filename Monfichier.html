WEB MOBILE • RESPONSIVE WEB DESIGN

Comprendre le viewport et les media queries dans la conception d’un site responsive

Cette page présente les principaux choix techniques mis en œuvre dans le cadre de l’évaluation de responsive web design.

Enseignant : Damien Bouchard
Réalisé par : Audrey Brun

Contexte du projet

Ce travail s’inscrit dans le cadre du cours Web mobile et porte sur la réalisation d’un site responsive. L’objectif est de montrer comment une même page web peut s’adapter à plusieurs formats d’écran, tout en conservant une bonne lisibilité, une organisation cohérente des contenus et une expérience utilisateur satisfaisante.

Dans cette page, l’accent est mis sur deux éléments fondamentaux du responsive design : la directive viewport et les media queries. Ces outils permettent d’ajuster l’affichage selon l’appareil utilisé et de structurer une page capable de fonctionner aussi bien sur smartphone que sur écran de bureau.

Le viewport

Le viewport correspond à la zone visible de la page dans le navigateur. Sur un téléphone mobile, cette zone doit être correctement définie pour que la page s’affiche à la bonne échelle et ne soit pas réduite automatiquement.

La balise viewport permet au navigateur d’interpréter la largeur de l’écran comme référence d’affichage. Grâce à elle, le site n’apparaît pas comme une version miniature de la page bureau. Elle constitue donc une base essentielle pour rendre un site lisible et exploitable sur mobile.

Les media queries

Les media queries sont des règles CSS conditionnelles qui permettent de modifier la mise en page en fonction de la largeur de l’écran. Elles servent à adapter l’organisation des blocs, la disposition des colonnes, la taille des images ou encore l’espacement entre les éléments.

Dans une logique responsive, elles rendent possible une lecture plus confortable : sur mobile, les contenus s’empilent généralement sur une seule colonne ; sur tablette ou sur bureau, ils peuvent être répartis sur plusieurs colonnes afin d’utiliser l’espace de manière plus efficace.

Quatre morceaux de code commentés

1. La balise viewport

<!-- Le navigateur doit adapter la page à la largeur réelle de l’écran -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Ce que fait le code : il indique au navigateur que la largeur de la page doit correspondre à celle de l’écran utilisé.

Pourquoi il est utilisé : il évite que la page apparaisse trop petite sur smartphone.

Effet sur mobile et bureau : sur mobile, le contenu est lisible dès l’ouverture ; sur bureau, l’affichage reste stable et cohérent.

2. Les images responsives

/* L’image devient un bloc */
/* Elle ne dépasse jamais la largeur disponible */
/* Sa hauteur s’adapte automatiquement */
img {
  display: block;
  max-width: 100%;
  height: auto;
}

Ce que fait le code : il empêche les images de dépasser leur conteneur et conserve leurs proportions.

Pourquoi il est utilisé : il évite les débordements et améliore l’affichage sur petit écran.

Effet sur mobile et bureau : sur mobile, l’image se réduit correctement ; sur bureau, elle garde une taille adaptée sans déformation.

3. La grille en une colonne par défaut

/* On utilise une grille pour organiser les cartes */
/* Par défaut, il n’y a qu’une seule colonne */
/* On laisse un espace entre les blocs */
.code-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1.5rem;
}

Ce que fait le code : il organise les cartes de contenu sur une seule colonne par défaut.

Pourquoi il est utilisé : cette disposition est plus simple à lire sur mobile et s’inscrit dans une logique mobile-first.

Effet sur mobile et bureau : sur mobile, les blocs s’empilent naturellement ; sur bureau, cette base pourra être modifiée par une media query.

4. La media query à partir de 768 px

/* Si l’écran mesure au moins 768 pixels de large */
/* alors la grille passe de une à deux colonnes */
@media screen and (min-width: 768px) {
  .code-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

Ce que fait le code : il modifie la grille lorsque l’écran atteint au moins 768 pixels de large.

Pourquoi il est utilisé : il permet d’utiliser davantage l’espace disponible sur tablette et sur ordinateur.

Effet sur mobile et bureau : sur mobile, les cartes restent empilées ; sur écran plus large, elles s’affichent en deux colonnes.

Conclusion

Le viewport et les media queries sont deux outils essentiels du responsive design. Le premier définit correctement l’espace d’affichage, tandis que les secondes permettent d’ajuster la mise en page selon la largeur disponible.

Ensemble, ils garantissent une lecture plus fluide, une meilleure hiérarchie des contenus et une navigation adaptée à tous les supports.
